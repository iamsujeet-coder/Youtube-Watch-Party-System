# MERN YouTube Watch Party

Production-style MERN project with role-based real-time synced YouTube playback.

## Stack

- MongoDB + Mongoose
- Express + Node.js
- React (Vite) + Tailwind CSS
- Socket.IO (real-time sync)

## Folder structure (company-style MERN)

```text
project/
  client/        # React + Tailwind frontend
  server/        # Express + Socket.IO + Mongoose backend
  package.json   # root scripts (run both apps)
  .gitignore
  README.md
```

`node_modules` is auto-generated by npm and not part of source architecture.

## Setup

1. Install root dependencies:

```bash
npm install
```

2. Install workspace dependencies:

```bash
npm --prefix server install
npm --prefix client install
```

3. Environment setup:
   - Copy `server/.env.example` to `server/.env`
   - Copy `client/.env.example` to `client/.env`

4. Run full project (frontend + backend):

```bash
npm run dev
```

Frontend: `http://localhost:5173`  
Backend: `http://localhost:5000`

## Implemented features

- Create/join room with unique room code
- Host, Moderator, Participant roles
- Role-based action protection on backend
- Real-time synced `play`, `pause`, `seek`, `change_video`
- Host can assign roles and remove participants
- Responsive UI with Tailwind

## Note

If MongoDB is not connected, backend still runs in memory mode for development.

## Deployment (Render)

### Backend (Web Service)

- Root directory: `server`
- Build command: `npm install`
- Start command: `npm start`
- Environment variables:
  - `MONGO_URI=<your-atlas-uri>`
  - `CLIENT_URL=<your-frontend-url>`

### Frontend (Static Site)

- Root directory: `client`
- Build command: `npm install && npm run build`
- Publish directory: `dist`
- Environment variables:
  - `VITE_SERVER_URL=<your-backend-url>`

Free-tier note: backend may take a little time on first request after idle.
